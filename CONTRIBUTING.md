# 贡献指南

欢迎贡献。本项目的核心价值在于把模拟/混合信号 IC 全流程中来之不易的工程经验沉淀下来，
供后续使用者参考。如果你在使用中遇到了新的问题、找到了更好的解决方案，或写了有用的脚本，
非常欢迎以 Issue 或 Pull Request 的形式补充进来。

## 可以贡献什么

- **新的问题与解决方案**：补充到 [`docs/02_常见问题与排错.md`](docs/02_常见问题与排错.md)
  的对应分类中，沿用「现象/报错 → 原因 → 解决」的格式。
- **PDK / 工艺相关的约定**：补充到 [`docs/03_CSMC05_PDK备忘.md`](docs/03_CSMC05_PDK备忘.md)
  或 [`skills/csmc-pdk`](skills/csmc-pdk/SKILL.md)。
- **流程改进与新工具**：放入 [`tools/`](tools/)，并在相应文档中说明用法。
- **文档订正**：错别字、失效链接、表述不清之处。

## 提交前请确认

1. **不包含任何敏感信息。** 这是最重要的一条。请勿提交真实的服务器 IP、端口、学号、
   密码、VNC 显示号或内网主机名。所有此类值都应以占位符（如 `<服务器>`、`<学号>`、
   `<grade>`、`{hostname}`）表示，真实值只存在于本地的 `.env` 与 `*.local.json`
   （二者已被 `.gitignore` 忽略）。提交前请检查 `git diff`。
2. **语言风格保持专业、严谨。** 文档以陈述事实与操作步骤为主，避免口语化表达、比喻与
   表情符号。
3. **保留可追溯性。** 记录问题时尽量附上原始报错文本与触发条件，便于后来者按关键字检索。
4. **引擎改动需谨慎。** 桥接引擎 `src/apilot/` 是基于 Cadence 公开 SKILL IPC 机制的独立
   实现；改动后请在服务器上实测 `apilot start` / `apilot status` 与 `execute("1+2")` 通路。

## 流程

1. Fork 本仓库并创建特性分支：`git checkout -b feature/your-topic`。
2. 提交改动，撰写清晰的 commit message。
3. 发起 Pull Request，在描述中说明改动内容与动机；若是新增的问题记录，请简述触发条件。

提交即表示你同意你的贡献以本项目的 [GPL-3.0](LICENSE)（含第 7(b) 条署名条款）发布。
