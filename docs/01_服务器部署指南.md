# 01 · 服务器部署指南

本文档说明如何从零配置环境：本地安装、连接服务器、在 Virtuoso 中执行首条 SKILL。全程
沿用本项目的默认配置（已针对微电子学院服务器调整）。

> 约定：下文凡 `<尖括号>` 均为**占位符**，请替换为实际值。请勿将真实 IP、端口、学号、
> 密码写入任何公开文件或截图。

---

## 0. 前提条件

1. **校园网**：服务器为校内地址（如 `10.x.x.x` 网段），需在校园网或 VPN 内可达；无需
   跳板机，本项目默认直连。
2. **可登录服务器**：`ssh <学号>@<服务器>` 可登录（首次使用密码）。
3. **Virtuoso 正在运行**：SKILL 模式要求服务器上有一个正在运行的 Virtuoso 进程；GUI 需
   在 VNC 桌面中启动（命令行无法启动 GUI）。
4. **本地 Python ≥ 3.9**。

> 服务器上的工具版本（供参考，以实际为准）：Virtuoso **IC618 / 6.1.8-64b**、Spectre
> （`SPECTRE231`）、Calibre 2024，工艺为 **CSMC 0.5µm（库名 `st02`，PDK 目录
> `CSMC05_M3`）**，操作系统多为 CentOS 7。

## 1. 本地安装（一次性）

建议使用独立虚拟环境，避免安装至系统或 conda 的全局 Python。

```bash
git clone https://github.com/GuoJiacheng0402/analog-pilot.git
cd analog-pilot
python3 -m venv .venv
source .venv/bin/activate
pip install -e .

# 验证
apilot --help
python -c "from apilot import SkillClient; print('ok')"
```

> **注意（conda 用户）**：若 `import apilot` 报 `ModuleNotFoundError`，通常是因为
> 终端默认 `conda activate base`，editable 安装写入的 `.pth` 未被 anaconda 的 site.py
> 处理。
> - 临时方案：`export PYTHONPATH=$(pwd)/src`
> - 根治方案：注释掉 shell 配置中的 `conda activate base`；或定义别名一次性执行
>   `conda deactivate && source .venv/bin/activate`。

## 2. 配置连接（`.env`）

```bash
cp .env.example ~/.apilot/.env
$EDITOR ~/.apilot/.env
```

至少填写以下两项，其余保持模板默认值：

```dotenv
APILOT_REMOTE_HOST=<服务器地址>
APILOT_REMOTE_USER=<学号>

# 模板中已包含的 SCUT 必要项（请确认存在）：
APILOT_DISABLE_CONTROL_MASTER=true                                  # 必须
APILOT_REMOTE_SCRATCH_ROOT=/SM01/home/<grade>/<学号>/.apilot   # 避免使用 /tmp
```

> **为何 `APILOT_DISABLE_CONTROL_MASTER=true` 必须开启？**
> 若不关闭，桥接探测远端 Python 解释器时会返回空字符串，报
> `No Python interpreter found ... Detection output: ''`，导致 `start` 失败。关闭后仅每次
> 新会话首次握手增加 1~2 秒。原理与排查见 [02 · 常见问题与排错](02_常见问题与排错.md)。
>
> **为何 scratch 不使用 `/tmp`？** 服务器重启后 `/tmp` 会被清空，导致已部署的 daemon
> 文件丢失。指向家目录下的固定目录即可，桥接首次启动时会自动创建。

`.env` 的查找顺序：当前目录 `./.env` → `~/.apilot/.env`；命令行 `--env FILE`
优先级最高。

## 3. 配置 SSH 免密登录（一次性）

```bash
ssh-copy-id <学号>@<服务器>          # 输入一次密码，出现 "Number of key(s) added: 1" 即成功
ssh -o BatchMode=yes <学号>@<服务器> echo ok   # 不再提示密码、直接输出 ok 即成功
```

可选：在本地 `~/.ssh/config` 中配置别名，并保持长连接：

```sshconfig
Host edalab
    HostName <服务器地址>
    User <学号>
    ServerAliveInterval 30
    ServerAliveCountMax 4
```

> 撤销免密：登录服务器，编辑 `~/.ssh/authorized_keys` 删除对应公钥行即可。

---

## 4. 启动桥接

```bash
source .venv/bin/activate     # 需先激活 venv（conda 用户可能还需 export PYTHONPATH）
apilot start
```

`start` 执行三项操作：① 建立 SSH 隧道；② 将 SKILL daemon 相关文件部署至服务器的 scratch
目录；③ 探测远端 Python。成功后会**输出一行 `load("....il")`**，请记录该行。

---

## 5. 在 Virtuoso CIW 中加载 daemon

在 VNC 桌面的 Virtuoso CIW（命令解释窗口）中，粘贴上一步输出的命令：

```skill
load("/SM01/home/<grade>/<学号>/.apilot/.../apilot_setup.il")
```

CIW 输出类似 `[apilot] bridge ready: bind=...` 即表示成功。

> 如忘记该命令，再次执行 `apilot status` 会重新输出。

### 配置自动加载（推荐）

将 [`templates/cdsinit_autoload.il`](templates/cdsinit_autoload.il) 中的片段**追加**至
服务器的 `~/.cdsinit`（建议先备份：`cp ~/.cdsinit ~/.cdsinit.bak.$(date +%F)`）。此后每次
启动 Virtuoso 都会自动加载 daemon，无需手动粘贴。该片段以 `errset(... t)` 包裹，即使
setup 文件暂时缺失也不会阻塞 Virtuoso 启动。

> 注意：**当前已在运行的** Virtuoso 不会自动重载，仅在下次重启 Virtuoso 时生效；本次仍
> 需手动粘贴一次。

## 6. 验证

```bash
apilot status
# 期望输出：
#   [tunnel]  running
#   [daemon]  OK            （NO RESPONSE → 见下方排查）

# 或者一次性自检全链路（ssh / 隧道 / daemon / SKILL / spectre 通道）：
apilot selftest
# 期望全部 [ok]：
#   [ok] ssh reachable / tunnel / daemon / SKILL execute(1+2) / spectre channel
```

```python
from apilot import SkillClient
client = SkillClient.from_env()
print(client.execute("1+2"))            # SkillResult(status=SUCCESS, output='3')
```

> `[daemon] NO RESPONSE` 最常见的原因是 **VNC 桌面中 Virtuoso 弹出了未关闭的对话框**，
> 阻塞了 SKILL 通道；其次是未粘贴 load 命令、或 Virtuoso 未运行。

> **CIW 输出与返回值的区别**：`execute_skill()` 将结果返回给 Python，但**不会**打印在 CIW
> 窗口。如需同时在 CIW 显示，应显式添加 `printf`：
> `client.execute_skill(r'let((v) v=1+2 printf("1+2 = %d\n" v) v)')`。

## 7. 在服务器上手动启动 Virtuoso（日常）

若需自行在 VNC 中启动 Virtuoso：

```bash
cd ~/Design/CSMC05_M3          # 设计工作目录（已 attach st02 工艺）
source /SM01/eda/env_set/bashrc_cds       # Cadence 环境（顺序固定：先 cds）
source /SM01/eda/env_set/bashrc_mentor    # Calibre 环境（后 mentor，依赖 cds 已 source）
virtuoso &
```

> **注意**：直接运行 `virtuoso` 会报 `error while loading shared libraries: libap_sh.so`，
> 必须先 source 上述两个环境脚本，缺一不可。
>
> **注意**：`bashrc_cds` 会设置 `LD_PRELOAD`（指向旧版 OpenSSL，IC618 依赖），会影响该
> shell 中 git/curl/openssl 的行为。建议用于启动 Virtuoso 的 shell 在 source 后专用于
> 启动 Virtuoso，其他操作另开终端；并且**不应**将这两条 source 写入 `~/.bashrc`（会影响
> 所有登录会话，也会干扰桥接的免密探测）。

## 8. 多服务器（multi-profile，按需）

存在多台服务器时，可在 `.env` 中为第二台添加 profile 后缀（大小写敏感），以 `-p` 切换：

```dotenv
APILOT_REMOTE_HOST_lab2=<第二台服务器>
APILOT_REMOTE_USER_lab2=<学号>
APILOT_DISABLE_CONTROL_MASTER_lab2=true
APILOT_REMOTE_SCRATCH_ROOT_lab2=/SM01/home/<grade>/<学号>/.apilot
```

```bash
apilot start  -p lab2
apilot status -p lab2
```

```python
from apilot.spectre import SpectreRunner
sim = SpectreRunner.from_env(profile="lab2")
```

> 免密以「本地公钥 → 服务器」配置，第二台若为同账号、同公钥，通常无需再次 `ssh-copy-id`。

## 9. 备份

服务器上的设计可能因误操作、重装或配额清理而丢失，建议养成增量备份习惯：

```bash
BACKUP=~/ic-backup-$(date +%Y-%m-%d)
mkdir -p "$BACKUP"
rsync -avz <学号>@<服务器>:'~/Design/CSMC05_M3' "$BACKUP/"
rsync -avz <学号>@<服务器>:'~/.cdsinit' "$BACKUP/"
rsync -avz <学号>@<服务器>:'~/cds.lib'  "$BACKUP/"
```

更系统的「小步可回退 + 三层备份」做法见 [06 · 协作与迭代约定](06_协作与迭代约定.md)。
