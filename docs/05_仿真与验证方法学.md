# 05 · 仿真与验证方法学

本文档关注「如何使测量结果正确」，而非「如何使仿真运行」。以下方法在课程设计中反复使用，
且可迁移至其他电路。

---

## 1. 绕开 maestro，直接使用 Spectre netlist

maestro 的 GUI 对话框容易导致 SKILL daemon 阻塞（见 [常见问题与排错 B 节](02_常见问题与排错.md)）。
当任务涉及批量扫参、反复重跑或自动解析时，**直接编写 Spectre netlist，通过
`SpectreRunner` 提交并自行解析 PSF**，效率显著高于与 maestro 反复交互：

- 并行 AC + tran 单轮约 30s（maestro 约 90~120s）。
- 批量扫参时可并发提交多个作业；一次实测中约 25 个并发 Spectre 作业未触发 license 排队，
  墙钟时间由串行的十余分钟压缩至并行的一分钟以内。

> netlist 的编写、提交与 PSF 解析见 [`skills/apilot/SKILL.md`](../skills/apilot/SKILL.md)；
> 并行可通过多个并发连接 / 批量提交实现。

## 2. 运放：使用开环 AC 测量 Adc / PM / GBW

### 为何不能仅依赖闭环
闭环残差估算只能给出 Adc 的**下限**，无法测得真实的 PM / GBW，因此必须采用开环测量。

### Middlebrook 式 1G H / 1G F 注入
在反馈环路中串入一个**直流短路、交流开路**的大电感 `L=1GH`，并用一个**直流开路、交流
耦合**的大电容 `C=1GF` 注入交流激励。其效果为：
- 直流：环路闭合，工作点正确建立；
- 交流：环路在注入点断开，`vout/vac = -A_v(s)`，由此得到环路增益。

读取相位时，在 GBW 处可直接读出 PM（该接法下相位已含 180°，无需再加 180°）。

### 解析时 AC 信号为复数
解析 PSF 时，AC 节点为 Python 的 `complex` 类型，直接 `abs()` 取幅度、`atan2()`（或
`phaseDegUnwrapped`）取相位即可。相位应使用 unwrapped 形式（见
[PDK 备忘 §7](03_CSMC05_PDK备忘.md#7-输出表达式--测量)）。

## 3. BGR：使用启动瞬态测量温度系数（TC）

### 为何纯 DC 温扫会得到无效结果
BGR 无显式启动电路时，纯 DC 温扫容易收敛至**低压零电流分支**（VREF≈几十至几百 mV），此时
TC 表现为数千 ppm/°C。这并非电路故障，而是仿真落入了错误的工作点。

### 正确做法：逐点上电
对每个温度点：
1. 将 VDD 从 `0V` ramp 至额定（如 `rise=2µs`）；
2. transient 运行至稳态（如 `stop=80µs`）；
3. 取**末段**（如末 `10µs`）平均作为该温度点的 Vref；
4. **逐点独立上电，不读取上一点的 OP**。

```
TC = ΔVref / (mean(Vref) × ΔT) × 1e6      # 单位 ppm/°C
```

配合收紧的数值精度（`reltol=1e-6  vabstol=1e-9  iabstol=1e-15`），mV 级测量方不被数值
噪声污染。本项目 `tools/postlayout-verify/bgr_verify.py` 已实现该启动瞬态温扫。

## 4. 一个可迁移的方法：参数解耦

调整 BGR 温度系数时，存在两个**正交的控制量**：
- **bias scale**（电阻整体同步缩放）—— 主要控制 Vref(T) 抛物线的**曲率 |a|**；
- **电阻比例**（如 R1/R3 之比）—— 主要控制抛物线的**顶点位置**。

方法：先用 bias scale 将曲率 |a| 调至接近零，再用电阻比例将顶点移至目标温区中央。借助
这种「先曲率、后顶点」的解耦，可在**不改变拓扑**的前提下将 TC 改善约一个数量级。

> 该方法的普适价值在于：**先识别彼此正交的控制量，再分别调整**。它适用于许多「单一指标
> 受多个参数耦合影响」的调参问题，不限于 BGR。

---

## 5. 何时改拓扑：先穷尽同拓扑搜索空间

调参遇到瓶颈时，应先确认当前拓扑下已扫描了哪些维度、尚有哪些未扫描，再决定是否改动拓扑。
瓶颈往往源于尚未扫描的控制量，而非拓扑本身的极限。改拓扑代价大、风险高，应作为穷尽同
拓扑搜索之后的最后手段。

## 6. 测量前的自检清单

- [ ] 当前 simulator 确为 Spectre？（ADE 标题含 "spectre" 不代表实际运行 Spectre，需在 Setup 中确认 run dir 为 `.../spectre/...`）
- [ ] modelFiles 配置了 `section=tt` 与 `biptypical`（且未重复 include `bip`）？
- [ ] MOS 设置了 `fw`？电阻通过 `segL/segW` 定值？
- [ ] 测量 mV 级信号或 TC 时收紧了 `reltol/vabstol/iabstol`？
- [ ] 相位使用了 `phaseDegUnwrapped`？
- [ ] BGR 使用启动瞬态而非纯 DC 温扫？
- [ ] 后仿 testbench 的端口顺序与 PEX 子电路一致？
